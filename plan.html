Core Backend
Foundation
▾
What you're building
🔗
POST /shorten endpoint
Accept a URL, generate a unique 6–8 char base62 code, store in DB, return short URL.
Backend
↪️
GET /:code redirect
Look up the code in the DB, return HTTP 301 redirect to the original URL. 404 if not found.
Backend
🗃️
Database schema
Table: links(code TEXT PK, original_url TEXT, created_at TIMESTAMP, clicks INT)
Database
🔀
Collision-safe code generation
Generate random base62 string, check DB for collision, retry if taken. ~56B combos at 6 chars.
Backend
🛡️
URL validation
Reject invalid URLs before storing. Use a URL parsing library, not regex.
Backend

Recommended Deploy Stack

🟢 Easiest (Node.js)

  • Express or Hono backend
  • PostgreSQL on Neon (free)
  • Deploy on Railway
  • Frontend on Vercel

🐍 Python Option

  • FastAPI backend
  • PostgreSQL or SQLite
  • Deploy on Render
  • Frontend on Netlify

⚡ Full-stack (Next.js)

  • API routes = backend
  • Vercel Postgres or PlanetScale
  • Single deploy on Vercel
  • Easiest domain setup

🦀 Edge (Fast)

  • Cloudflare Workers
  • KV or D1 storage
  • Globally distributed
  • Very generous free tier
Request Flow
User → POST /shorten { url } ↓ Backend → validate URL → generate code (e.g. "x7kP2q") → INSERT into DB → return { short: "https://yo.ur/x7kP2q" } Visitor → GET /x7kP2q ↓ Backend → SELECT original_url WHERE code='x7kP2q' → UPDATE clicks = clicks + 1 → HTTP 301 Redirect → original URL
Start here: Get Phase 1 working locally first — a single Express/FastAPI file with SQLite is enough. Once redirects work end-to-end, move to Phase 2 and 3. Don't over-engineer early; the core is just a key-value lookup with a redirect.
